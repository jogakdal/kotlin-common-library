# 별첨: 타 라이브러리 비교

Excel 보고서 생성 라이브러리를 선택할 때 참고할 수 있도록, TBEG과 유사한 템플릿 기반 라이브러리인 JXLS, JETT, ExcelReportGenerator(ERG)를 비교합니다.

---

## 1. 비교 대상

### JXLS

JVM 생태계에서 가장 널리 알려진 템플릿 기반 Excel 생성 라이브러리입니다. Apache POI 기반으로 동작하며, **엑셀 셀의 노트(Comment)** 에 명령어를 작성하여 보고서를 구성합니다.

- **플랫폼**: JVM (Java 17)
- **Excel 엔진**: Apache POI
- **라이선스**: Apache License 2.0
- **GitHub**: [jxlsteam/jxls](https://github.com/jxlsteam/jxls)

### JETT (Java Excel Template Translator)

JVM 기반의 템플릿 방식 Excel 생성 라이브러리입니다. Apache POI 기반으로 동작하며, **XML 태그 문법**(`<jt:forEach>`, `<jt:if>` 등)을 셀에 작성하여 보고서를 구성합니다. JEXL(Java Expression Language) 기반 표현식과 jAgg 라이브러리를 활용한 집계 기능을 제공합니다.

- **플랫폼**: JVM (Java 7+)
- **Excel 엔진**: Apache POI
- **라이선스**: LGPL v3
- **GitHub**: [rgettman/jett](https://github.com/rgettman/jett)

> [!WARNING]
> JETT는 2018년경 이후 업데이트가 중단된 상태입니다. Apache POI 5.x 및 Java 21 환경에서의 호환성이 검증되지 않았습니다.

### ExcelReportGenerator (ERG)

.NET 기반의 Excel 보고서 생성 라이브러리입니다. ClosedXML을 기반으로 동작하며, Named Range와 "패널" 계층 구조를 통해 보고서를 구성합니다.

- **플랫폼**: .NET Standard 2.0
- **Excel 엔진**: ClosedXML (OpenXML 래퍼)
- **라이선스**: MIT
- **GitHub**: [traf72/excel-report-generator](https://github.com/traf72/excel-report-generator)

### TBEG (Template-Based Excel Generator)

Kotlin/JVM 기반의 Excel 보고서 생성 라이브러리입니다. Excel 템플릿 셀에 마커를 직접 작성하는 방식으로, 디자이너가 만든 Excel 양식을 그대로 활용하면서 데이터만 바인딩하면 됩니다.

- **플랫폼**: JVM (Kotlin/Java)
- **Excel 엔진**: Apache POI
- **패키지**: `com.hunet.common:tbeg`

### 3.5 차트 및 피벗 테이블

| | ERG | JXLS | JETT | TBEG |
|---|---|---|---|---|
| **차트 데이터 범위 조정** | 미지원 | 미지원 (동적 이름 범위로 우회 필요) | 미지원 (기존 차트 유지만 가능) | 자동 조정 |
| **피벗 테이블 소스 범위 조정** | 미지원 | 미지원 (파일 열 때 새로고침 필요) | 미지원 | 자동 조정 |
| **파일 열 때 새로고침 없이 사용** | - | X | - | O |

JXLS에서 차트를 사용하려면 Excel의 동적 이름 범위(`OFFSET` 수식 등)를 별도로 설정하여 데이터 확장에 대응해야 합니다. 피벗 테이블은 템플릿의 피벗 테이블 옵션에서 "파일을 열 때 데이터 새로고침"을 활성화해야 합니다.

TBEG은 차트의 데이터 범위와 피벗 테이블의 소스 범위를 생성 시점에 자동으로 조정하므로, 결과 파일을 열 때 별도의 새로고침이 필요 없습니다.

## 4. 타 라이브러리의 고유 강점

공정한 비교를 위해, 각 라이브러리가 가진 고유한 강점도 살펴보겠습니다.

### JXLS

**그룹화와 집계**

`jx:each`의 `groupBy` 속성과 `GroupSum` 클래스로 데이터 그룹화와 집계를 선언적으로 처리할 수 있습니다.

TBEG에서는 Excel 수식(`=SUM()`, `=AVERAGE()` 등)으로 동일한 결과를 얻을 수 있으며, 이 방식은 **Excel에서 직접 계산 로직을 확인하고 검증할 수 있다**는 장점이 있습니다.

**조건부 렌더링**

`jx:if` 명령어로 조건에 따라 영역을 표시하거나 숨길 수 있습니다. TBEG에서는 DataProvider에서 데이터를 사전 가공하여 동일한 결과를 달성할 수 있습니다.

**다중 시트 자동 생성**

`multisheet` 속성으로 컬렉션의 각 항목을 별도의 시트로 자동 생성할 수 있습니다.

### JETT

**JDBC 직접 쿼리**

템플릿 내에서 JDBC 쿼리를 직접 실행할 수 있어, 별도의 데이터 조회 코드 없이 데이터베이스에서 결과를 가져올 수 있습니다.

**jAgg 집계 엔진**

jAgg 라이브러리와 연동하여 Sum, Average, Rollup, Cube 등 다양한 집계 연산을 템플릿 레벨에서 수행할 수 있습니다. TBEG에서는 Excel 수식으로 동일한 결과를 얻을 수 있습니다.

**CellListener**

셀 단위 커스텀 처리(교대 행 하이라이팅 등)를 위한 리스너를 제공합니다.

### ERG

**빌트인 집계 함수**

Sum, Avg, Count, Min, Max와 사용자 정의 집계 함수를 패널에 내장하고 있습니다.

**자동 셀 병합**

GroupBy로 동일한 값을 가진 셀을 자동으로 병합합니다. 부서별 직원 목록처럼 같은 부서명이 반복되는 경우에 유용합니다.

**패널 계층 구조**

Parent/Children 패널 트리로 복잡한 중첩 보고서를 구성할 수 있으며, 패널별 Before/After 렌더링 훅으로 세밀한 제어가 가능합니다.

## 6. 적합한 선택 가이드

### TBEG이 적합한 경우

- **디자이너가 만든 Excel 양식**을 그대로 활용해야 할 때
- 보고서에 **이미지, 차트, 피벗 테이블**이 포함될 때
- **수식 기반의 자동 계산**이 중요한 보고서일 때
- **수만~수십만 행**의 대용량 데이터를 안정적으로 처리해야 할 때
- **비동기 처리**와 **진행률 확인**이 필요할 때
- **Spring Boot** 환경에서 간편하게 통합하고 싶을 때
- 템플릿 변경 시 **코드 수정 없이** 대응하고 싶을 때

### JXLS가 적합한 경우

- 조건부 렌더링(`jx:if`)이 핵심 요구사항일 때
- **다중 시트 자동 생성**이 필요할 때
- 텍스트 데이터 중심의 보고서일 때

### JETT가 적합한 경우

- JDBC 쿼리를 템플릿에서 직접 실행해야 할 때
- `<jt:if>` 등 조건부 렌더링이 필요할 때
- 단, 유지보수가 중단된 상태이므로 **신규 프로젝트에는 권장하지 않습니다**

### ERG가 적합한 경우

- **.NET** 기반 프로젝트일 때
- 코드에서 **집계와 그룹화**를 선언적으로 처리하고 싶을 때
- **패널 트리**로 복잡한 중첩 구조를 코드 레벨에서 제어해야 할 때

> [!TIP]
> TBEG은 빌트인 집계 기능 대신 **Excel 수식을 활용**하는 접근을 취합니다. `=SUM()`, `=AVERAGE()` 등의 수식을 템플릿에 미리 배치하면 데이터 확장 시 범위가 자동으로 조정되어, **Excel에서 직접 계산 로직을 확인하고 검증할 수 있다**는 장점이 있습니다.

## 7. 상용 솔루션 대비 TBEG의 장점

Aspose.Cells, GcExcel(Document Solutions for Excel) 등 상용 솔루션은 Excel 읽기/쓰기/변환을 모두 아우르는 범용 Excel 처리 라이브러리입니다. TBEG은 **보고서 생성**이라는 목적에 집중하여, 다음과 같은 차별점을 제공합니다.

### 비용

상용 솔루션은 개발자 라이선스 기준 $1,000 이상의 비용이 발생합니다. TBEG은 Apache 2.0 라이선스로 무료 사용이 가능합니다.

### 템플릿 중심 워크플로우

상용 솔루션은 **코드에서 Excel을 조작**하는 방식이 기본입니다. 셀 서식, 차트, 수식 등 모든 요소를 코드로 제어해야 합니다. TBEG은 디자이너가 만든 **Excel 템플릿을 그대로 활용**하고, 코드는 데이터 바인딩에만 집중합니다. 템플릿 변경 시 코드 수정이 필요 없습니다.

### Spring Boot 네이티브 통합

상용 솔루션은 프레임워크 통합을 별도로 구성해야 합니다. TBEG은 의존성 추가만으로 AutoConfiguration, 설정 파일 연동, 비동기 처리(Coroutines/CompletableFuture), 진행률 콜백이 바로 동작합니다.

### 경량성

상용 솔루션은 독자 엔진을 내장하여 JAR 크기가 크고, Excel 파일의 읽기/쓰기/변환까지 포함하는 범용 라이브러리입니다. TBEG은 보고서 생성에 필요한 기능만 포함하며, Apache POI 기반으로 기존 POI 코드와의 호환성도 유지됩니다.

### 상용 솔루션이 적합한 경우

다음과 같은 요구사항이 있다면 상용 솔루션이 더 적합할 수 있습니다.

- Excel 파일의 **읽기/파싱**이 필요한 경우
- Excel을 **PDF, HTML, 이미지로 변환**해야 하는 경우
- 코드에서 Excel의 **모든 요소를 동적으로 생성/제어**해야 하는 경우
- **자체 수식 계산 엔진**이 필요한 경우 (서버에서 수식 결과를 미리 계산)
