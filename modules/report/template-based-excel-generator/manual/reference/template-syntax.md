## 1. 마커 개요

### 1.1 마커란

**마커**는 Excel 템플릿에서 데이터가 들어갈 자리를 표시하는 특수한 텍스트입니다. TBEG은 템플릿을 처리할 때 마커를 실제 데이터로 치환하고, 정상적으로 처리된 마커는 결과 파일에 나타나지 않습니다.

### 1.2 기본 형식

모든 마커는 `${...}` 형식으로 Excel 셀에 작성합니다.

```
${...}
```

마커는 **변수형 마커**와 **함수형 마커** 두 가지로 나뉩니다.

#### 변수형 마커

`${변수명}` 형태로, 데이터의 값으로 그대로 치환됩니다. 점(`.`) 표기법으로 객체의 프로퍼티에 접근할 수 있으며, 여러 단계로 중첩된 필드도 지원합니다.

```
${title}                  // 단순 값 치환
${emp.name}               // 객체 프로퍼티 접근
${emp.department.name}    // 중첩 프로퍼티 접근
보고서: ${title}           // 텍스트와 조합
```

셀 값뿐 아니라 수식 인자, 차트 제목, 도형 텍스트, 머리글/바닥글에서도 사용할 수 있습니다. 바인딩하는 값이 `=`로 시작하면 Excel 수식으로 처리됩니다 ([2.3 수식 바인딩](#23-수식-바인딩) 참조).

#### 함수형 마커

`${함수(파라미터...)}` 형태로, 데이터를 기반으로 특정 동작을 수행합니다.

| 함수 | 역할 | 예시 |
|------|------|------|
| `repeat` | 컬렉션 데이터를 행/열로 반복 확장 | `${repeat(employees, A2:C2, emp)}` |
| `image` | 동적 이미지 삽입 | `${image(logo, B2, 200:150)}` |
| `size` | 컬렉션의 아이템 수 출력 | `${size(employees)}` |
| `merge` | 연속된 같은 값의 셀 자동 병합 | `${merge(emp.dept)}` |
| `bundle` | 여러 요소를 하나의 단위로 묶음 | `${bundle(A5:H12)}` |

#### 대소문자 구분

| 요소 | 구분 여부 | 예시 |
|------|----------|------|
| 함수명 | 구분 안 함 | `${Repeat(...)}`, `${REPEAT(...)}`, `=tbeg_repeat(...)` 모두 동일 |
| 키워드 인자 (`direction`, `size`) | 구분 안 함 | `DOWN`, `down`, `Right`, `FIT`, `Original` 모두 동일 |
| 파라미터 이름 (명시적 형식) | 구분 안 함 | `Collection=`, `RANGE=`, `var=` 모두 동일 |
| 데이터 키 (컬렉션명, 변수명, 필드명, 이미지명) | **구분함** | `${employeeName}`과 `${employeename}`은 다름 |

#### 수식 형태

함수형 마커는 `=TBEG_함수(파라미터...)` 형식의 **수식 형태**로도 작성할 수 있습니다. 동작은 동일하며, Excel의 셀 참조 기능(클릭, 드래그 선택 등)을 활용하여 범위를 지정할 수 있습니다.

| 텍스트 형태 | 수식 형태 |
|---|---|
| `${repeat(col, range, var)}` | `=TBEG_REPEAT(col, range, var)` |
| `${image(name)}` | `=TBEG_IMAGE(name)` |
| `${size(col)}` | `=TBEG_SIZE(col)` |
| `${merge(item.field)}` | `=TBEG_MERGE(item.field)` |
| `${bundle(범위)}` | `=TBEG_BUNDLE(범위)` |

> [!NOTE]
> 수식 형태 마커는 Excel에서 `#NAME?` 에러로 표시됩니다. 이는 정상이며, 생성 시 올바르게 처리됩니다.

### 1.3 함수형 마커 공통 규칙

다음 규칙은 모든 함수형 마커에 공통으로 적용됩니다.

#### 마커 배치

함수형 마커는 대상 범위 밖이라면 워크북 내 어디에든 배치할 수 있습니다. 다른 시트에 배치해도 됩니다.

예를 들어 `${repeat(employees, A2:C2, emp)}`는 반복 범위 A2:C2 밖이라면 같은 시트의 어느 셀이든, 다른 시트이든 상관없이 배치할 수 있습니다. `${bundle(A5:H12)}`도 bundle 범위 밖이라면 어디에든 배치 가능합니다.

#### 명시적 파라미터 형식

모든 함수형 마커의 파라미터에 이름을 명시적으로 지정할 수 있습니다.

```
// 위치 기반
${repeat(employees, A2:C2, emp)}
${image(logo, B2, 200:150)}

// 명시적
${repeat(collection=employees, range=A2:C2, var=emp)}
${image(name=logo, position=B2, size=200:150)}
```

수식 형태에서도 동일하게 사용할 수 있습니다.

```
=TBEG_REPEAT(collection=employees, range=A2:C2, var=emp)
=TBEG_IMAGE(name=logo, position=B2, size=original)
```

#### 파라미터 생략

명시적 형식에서 선택적 파라미터는 생략하거나, `NULL` 또는 빈 값으로 지정할 수 있습니다. 아래 세 가지는 모두 동일합니다.

```
${repeat(collection=items, range=A2:C2, empty=A10:C10)}           // var 생략
${repeat(collection=items, range=A2:C2, var=NULL, empty=A10:C10)} // var=NULL
${repeat(collection=items, range=A2:C2, var=, empty=A10:C10)}     // var= (빈 값)
```

#### 혼합 사용 불가

위치 기반과 명시적 파라미터는 혼합할 수 없습니다. 하나의 파라미터라도 이름을 명시하면 모든 파라미터에 이름을 명시해야 합니다.

```
// 올바른 예
${repeat(items, A2:C2, item, DOWN, A10:C10)}                                    // 모두 위치 기반
${repeat(collection=items, range=A2:C2, var=item, direction=DOWN, empty=A10:C10)} // 모두 명시적

// 잘못된 예 (오류 발생)
${repeat(items, A2:C2, item, empty=A10:C10)}         // 혼합 불가
${repeat(items, A2:C2, var=item, direction=DOWN)}   // 혼합 불가
```

#### 범위 경계 걸침 금지

셀 범위를 다루는 모든 요소(repeat 범위, bundle 범위, 병합된 셀 등)는 서로 부분적으로 걸치면 안 됩니다. 각 요소는 다른 요소 안에 완전히 포함되거나 완전히 밖에 있어야 합니다.

```
// 올바른 예
${repeat(a, A2:C2, ...)}    // A~C열
${repeat(b, E2:G2, ...)}    // E~G열 (완전히 분리)
${bundle(A5:G10)}            // 두 repeat 모두 완전히 포함

// 잘못된 예 (오류 발생)
${repeat(a, A2:C2, ...)}    // A~C열
${bundle(B5:G10)}            // B열에서 repeat 범위와 부분적으로 걸침
```

#### 위치 기반 파라미터 생략

위치 기반 형식에서 중간 파라미터를 생략하려면 빈 값으로 두면 됩니다.

```
// direction(4번째)을 생략하고 empty(5번째)만 지정
${repeat(items, A2:C2, item, , A10:C10)}

// var(3번째)와 direction(4번째)을 생략
${repeat(items, A2:C2, , , A10:C10)}

// image에서 position(2번째)을 생략하고 size(3번째)만 지정
${image(logo, , 200:150)}
```

## 2. 변수 치환

### 2.1 단순 변수

**문법**: `${변수명}`

템플릿의 `${변수명}` 마커를 데이터의 해당 키 값으로 치환합니다.

#### 템플릿

|   | A      | B         |
|---|--------|-----------|
| 1 | 제목     | ${title}  |
| 2 | 작성일    | ${date}   |
| 3 | 작성자    | ${author} |

#### 데이터

```kotlin
mapOf(
    "title" to "월간 보고서",
    "date" to "2026-01-15",
    "author" to "황용호"
)
```

#### 결과

|   | A   | B          |
|---|-----|------------|
| 1 | 제목  | 월간 보고서     |
| 2 | 작성일 | 2026-01-15 |
| 3 | 작성자 | 황용호        |

#### 지원 타입

점(`.`) 표기법으로 접근하는 대상은 다음 타입을 지원합니다.

- **객체 필드**: data class, POJO의 프로퍼티/필드
- **Map 키**: Map의 키로 접근
- **getter 메서드**: `getFieldName()` 형태의 getter

이 규칙은 단순 변수(`${emp.name}`)와 repeat 내 아이템 필드(`${item.field}`) 모두에 동일하게 적용됩니다.

### 2.2 복합 텍스트

하나의 셀에 여러 변수와 텍스트를 조합할 수 있습니다.

```
작성자: ${author} (${department})
```

### 2.3 수식 바인딩

바인딩하는 값이 `=`로 시작하면 텍스트가 아닌 **Excel 수식**으로 처리됩니다.

#### 기본 사용

```kotlin
val data = mapOf("formula" to "=SUM(A1:A10)")
// 템플릿 셀: ${formula}
// 결과: =SUM(A1:A10) (실제 수식)
```

이를 활용하면 데이터에 따라 수식을 동적으로 결정할 수 있습니다.

#### repeat 내 수식 바인딩

repeat 내 아이템 필드에서도 동일하게 동작합니다. repeat에 의한 수식 범위 자동 조정(확장, 행 시프트)도 적용됩니다.

**템플릿**

|   | A                                | B             | C             | D                |
|---|----------------------------------|---------------|---------------|------------------|
| 1 | 이름                               | 매출            | 목표            | 달성률              |
| 2 | ${s.name}                        | ${s.amount}   | ${s.target}   | ${s.rateFormula} |
| 3 | ${repeat(sales, A2:D2, s)}       |               |               |                  |
| 4 | 합계                               | ${totalRevenue} |             |                  |

**데이터**

```kotlin
mapOf(
    "sales" to listOf(
        mapOf("name" to "A팀", "amount" to 15000, "target" to 20000, "rateFormula" to "=B2/C2"),
        mapOf("name" to "B팀", "amount" to 22000, "target" to 18000, "rateFormula" to "=B2/C2"),
    ),
    "totalRevenue" to "=SUM(B2:B2)"
)
```

**결과** (2개 아이템)

|   | A  | B      | C      | D          |
|---|----|--------|--------|------------|
| 1 | 이름 | 매출     | 목표     | 달성률        |
| 2 | A팀 | 15,000 | 20,000 | =B2/C2     |
| 3 | B팀 | 22,000 | 18,000 | =B3/C3     |
| 4 | 합계 | =SUM(B2:B3) |   |            |

- `rateFormula`(`=B2/C2`)가 각 행에서 `=B2/C2`, `=B3/C3`으로 행 시프트됩니다
- `totalRevenue`(`=SUM(B2:B2)`)가 repeat 확장에 맞게 `=SUM(B2:B3)`으로 범위 확장됩니다

#### 주의사항

`=`로 시작하는 **모든** 문자열 값이 수식으로 처리됩니다. `"=A급 인재"`처럼 등호로 시작하는 일반 텍스트를 바인딩하면 유효하지 않은 수식으로 처리되어 오류가 발생할 수 있습니다. 등호로 시작하는 텍스트를 그대로 출력해야 하는 경우, 앞에 공백이나 아포스트로피를 추가하여 수식으로 인식되지 않도록 합니다.

```kotlin
// 수식으로 처리됨 (의도한 경우)
mapOf("formula" to "=SUM(A1:A10)")

// 텍스트로 유지하려면 앞에 공백 추가
mapOf("grade" to " =A급 인재")
```

#### 자동 숫자 서식

수식으로 치환된 셀의 표시 형식이 "일반"인 경우, 자동으로 정수 숫자 서식(`#,##0`)이 적용됩니다. 소수점이 필요한 수식(예: 비율, 평균)은 템플릿 셀에 원하는 표시 형식(예: `0.0%`, `#,##0.00`)을 직접 지정합니다.

> [!TIP]
> repeat 내 수식의 범위 자동 조정(행 시프트, 범위 확장)에 대한 상세 규칙은 [3.6 관련 요소 자동 조정](#36-관련-요소-자동-조정)을 참조하세요.

## 3. 반복 처리

### 3.1 기본 반복 (DOWN 방향)

**문법**: `${repeat(컬렉션, 범위, 변수)}`

컬렉션의 각 아이템에 대해 지정된 범위를 아래로 반복합니다.

#### 템플릿

|   | A                                  | B               | C             |
|---|------------------------------------|-----------------|---------------|
| 1 | ${repeat(employees, A3:C3, emp)}   |                 |               |
| 2 | 이름                                 | 직급              | 연봉            |
| 3 | ${emp.name}                        | ${emp.position} | ${emp.salary} |

#### 데이터

```kotlin
data class Employee(val name: String, val position: String, val salary: Int)

mapOf(
    "employees" to listOf(
        Employee("황용호", "부장", 8000),
        Employee("한용호", "과장", 6500),
        Employee("홍용호", "대리", 4500)
    )
)
```

#### 결과

|   | A   | B  | C     |
|---|-----|----|-------|
| 1 |     |    |       |
| 2 | 이름  | 직급 | 연봉    |
| 3 | 황용호 | 부장 | 8,000 |
| 4 | 한용호 | 과장 | 6,500 |
| 5 | 홍용호 | 대리 | 4,500 |

### 3.2 다중 행 반복

범위에 여러 행을 지정하면 다중 행이 함께 반복됩니다.

#### 템플릿

|   | A                                | B                   |
|---|----------------------------------|---------------------|
| 1 | ${repeat(employees, A2:B3, emp)} |                     |
| 2 | 이름: ${emp.name}                  | 직급: ${emp.position} |
| 3 | 급여: ${emp.salary}                |                     |

### 3.3 오른쪽 방향 반복 (RIGHT)

**문법**: `${repeat(컬렉션, 범위, 변수, RIGHT)}`

컬렉션의 각 아이템에 대해 지정된 범위를 오른쪽으로 반복합니다.

#### 템플릿

|   | A                                      | B           |
|---|----------------------------------------|-------------|
| 1 | ${repeat(months, B1:B2, m, RIGHT)}     | ${m.month}  |
| 2 |                                        | ${m.sales}  |

#### 결과

|   | A  | B      | C      | D      |
|---|----|--------|--------|--------|
| 1 |    | 1월     | 2월     | 3월     |
| 2 |    | 100    | 150    | 200    |

### 3.4 다중 반복 영역

한 시트에 여러 개의 반복 영역을 사용할 수 있습니다.

#### 열 그룹 분리

서로 다른 열에 위치한 반복 영역은 독립적으로 확장됩니다.

|   | A                                | B             | C | D                                   | E              |
|---|----------------------------------|---------------|---|-------------------------------------|----------------|
| 1 | ${repeat(employees, A2:B2, emp)} |               |   | ${repeat(departments, D2:E2, dept)} |                |
| 2 | ${emp.name}                      | ${emp.salary} |   | ${dept.name}                        | ${dept.budget} |

#### 중복 마커

같은 컬렉션과 같은 대상 범위를 가진 repeat 마커가 여러 개 있으면 중복으로 간주됩니다. 중복 마커가 발견되면 경고 로그가 출력되고 마지막 마커만 동작합니다.

- 다른 시트에 있어도 시트 접두사(`'Sheet1'!A2:C2`)로 같은 대상을 참조하면 중복입니다
- 같은 범위라도 컬렉션이 다르면 중복이 아닙니다

#### 제한사항

- 반복 영역은 다른 범위 요소와 부분적으로 걸치면 안 됩니다 ([범위 경계 걸침 금지](#범위-경계-걸침-금지) 참조)
- 같은 열 범위 내에서 여러 반복 영역을 세로로 배치할 수 있습니다

### 3.5 빈 컬렉션 처리 (empty)

**문법**: `${repeat(컬렉션, 범위, 변수, 방향, 대체범위)}`

컬렉션이 비어있을 때 대체 셀 범위의 내용을 표시합니다.

#### 기본 동작 (empty 미지정)

컬렉션이 비어있고 `empty` 파라미터가 없으면, 반복 영역 크기에 맞는 빈 행(DOWN 모드) 또는 빈 열(RIGHT 모드)이 1개 출력됩니다.

- **빈 행/열의 범위**: 엑셀 전체 행이 아닌 반복 영역(range)에 해당하는 셀들만 빈 값으로 출력됩니다.
- **예**: `range=A2:C2`인 경우 A, B, C 열의 셀만 빈 값이 되며, D열 이후는 영향받지 않습니다.

#### empty 파라미터 사용

컬렉션이 비어있을 때 지정된 셀 범위의 내용을 대신 표시합니다.

#### 템플릿

|     | A                                               | B               | C             |
|-----|-------------------------------------------------|-----------------|---------------|
| 1   | ${repeat(employees, A2:C2, emp, DOWN, A10:C10)} |                 |               |
| 2   | ${emp.name}                                     | ${emp.position} | ${emp.salary} |
| ... |                                                 |                 |               |
| 10  | (데이터가 없습니다)                                     |                 |               |

#### 데이터 (빈 컬렉션)

```kotlin
mapOf("employees" to emptyList<Employee>())
```

#### 결과

|   | A           | B | C |
|---|-------------|---|---|
| 1 |             |   |   |
| 2 | (데이터가 없습니다) |   |   |

> [!NOTE]
> - A10:C10의 내용과 스타일이 A2:C2 위치에 복사됩니다.
> - A10:C10 원본 셀은 결과 파일에서 빈 셀로 처리됩니다.
> - empty 범위가 단일 셀이고 반복 영역이 더 크면 자동으로 셀이 병합됩니다.

#### 단일 셀 병합 예제

empty 범위가 단일 셀이면 반복 영역 전체를 병합하여 해당 내용을 표시합니다.

**템플릿**

|     | A                                           | B               | C             |
|-----|---------------------------------------------|-----------------|---------------|
| 1   | ${repeat(employees, A2:C2, emp, DOWN, A10)} |                 |               |
| 2   | ${emp.name}                                 | ${emp.position} | ${emp.salary} |
| ... |                                             |                 |               |
| 10  | 데이터가 없습니다                                   |                 |               |

- **A10**: 단일 셀에 메시지 작성 (A10:C10이 아닌 A10만 지정)

**결과 (빈 컬렉션)**

<table>
  <tr><th></th><th>A</th><th>B</th><th>C</th></tr>
  <tr><td>1</td><td></td><td></td><td></td></tr>
  <tr><td>2</td><td colspan="3" style="text-align:center">데이터가 없습니다</td></tr>
</table>

- A2:C2 영역이 자동으로 병합되고 A10의 내용이 표시됩니다.

#### empty 범위 지정 방식

```
${repeat(items, A2:C3, item, DOWN, A10:C11)}     // 일반 범위
${repeat(items, A2:C3, item, DOWN, $A$10:$C$11)} // 절대 좌표 ($ 무시)
${repeat(items, A2:C3, item, DOWN, 'Sheet2'!A1:C1)} // 다른 시트 참조
```

#### 위치 파라미터 방식

명시적 파라미터 이름 없이 5번째 파라미터로 지정할 수도 있습니다.

```
${repeat(items, A2:C3, item, DOWN, A10:C11)}
=TBEG_REPEAT(items, A2:C3, item, DOWN, A10:C11)
```

### 3.6 관련 요소 자동 조정

반복 영역이 확장되면 영향 받는 Excel 요소들의 좌표와 범위가 자동으로 조정됩니다.

#### 자동 조정되는 요소

| 요소     | 조정 내용                                  |
|--------|----------------------------------------|
| 수식 참조  | 셀 참조(예: `=SUM(A1:A10)`)가 확장된 범위에 맞게 조정 |
| 차트     | 데이터 소스 범위가 확장된 데이터에 맞게 조정              |
| 피벗 테이블 | 소스 데이터 범위가 확장된 데이터에 맞게 조정              |
| 병합 셀   | 반복 영역 외부의 병합 셀 위치가 조정                  |
| 조건부 서식 | 적용 범위가 확장된 영역에 맞게 조정                   |

#### 위치 이동 및 범위 확장

반복 영역이 확장되면:
- 반복 영역 아래에 있는 요소는 확장된 만큼 아래로 이동합니다.
- 반복 영역을 참조하는 수식은 확장된 전체 영역을 참조하게 됩니다.

**템플릿**

|   | A                                | B             |
|---|----------------------------------|---------------|
| 1 | ${repeat(items, A2:B2, item)}    |               |
| 2 | ${item.name}                     | ${item.value} |
| 3 | 합계                               | =SUM(B2:B2)   |

**결과** (items가 3개인 경우)

|   | A   | B           |
|---|-----|-------------|
| 1 |     |             |
| 2 | 항목1 | 100         |
| 3 | 항목2 | 200         |
| 4 | 항목3 | 300         |
| 5 | 합계  | =SUM(B2:B4) |

> [!NOTE]
> 3행에 있던 합계 행이 5행으로 이동하고, 수식 `=SUM(B2:B2)`도 확장된 범위 `=SUM(B2:B4)`를 참조합니다.

#### 수식 참조 유형별 동작

| 참조 유형 | 예시 | 확장 동작 |
|----------|------|----------|
| 상대 참조 | `B3:B3` | 확장됨 (`B3:B5`) |
| 절대 참조 | `$B$3:$B$3` | 유지됨 (확장 안 함) |
| 행 절대 참조 | `B$3:B$3` | 유지됨 (DOWN 방향 확장 안 함) |
| 열 절대 참조 | `$B3:$B3` | 확장됨 (DOWN 방향 `$B3:$B5`) |
| 다른 시트 참조 | `Sheet2!B3:B3` | 해당 시트의 repeat 확장이 반영됨 |

**예시: 절대 참조로 확장 제어**

```
=SUM(B3:B3)      -> =SUM(B3:B5)     // 상대 참조: 확장됨
=SUM($B$3:$B$3)  -> =SUM($B$3:$B$3) // 절대 참조: 그대로 유지
=SUM($B3:$B3)    -> =SUM($B3:$B5)   // 열만 절대: 행 방향 확장됨
```

**예시: 다른 시트의 repeat 영역 참조**

Sheet2에 repeat 영역이 있고 3개의 항목으로 확장되면, Sheet1에서 Sheet2를 참조하는 수식도 함께 확장됩니다.

```
=SUM(Sheet2!B3:B3)       -> =SUM(Sheet2!B3:B5)
=SUM('Data Sheet'!B3:B3) -> =SUM('Data Sheet'!B3:B5)
```

## 4. 이미지 삽입

### 4.1 기본 이미지

**문법**: `${image(이름)}`

마커가 있는 셀(또는 병합 영역)에 이미지를 삽입합니다.

#### 템플릿

|   | A              | B               |
|---|----------------|-----------------|
| 1 | ${image(logo)} | 회사명: ${company} |
| 2 | (병합된 셀)        |                 |

#### 데이터

```kotlin
val provider = simpleDataProvider {
    value("company", "(주)휴넷")

    // ByteArray로 제공
    image("logo", logoBytes)

    // 또는 URL로 제공 (렌더링 시 자동 다운로드)
    imageUrl("logo", "https://example.com/logo.png")
}
```

### 4.2 위치 지정

**문법**: `${image(이름, 위치)}`

이미지를 삽입할 셀 위치를 직접 지정합니다.

```
${image(logo, B2)}      // B2 셀에 삽입
${image(stamp, D5:F8)}  // D5:F8 범위에 삽입
```

### 4.3 크기 지정

**문법**: `${image(이름, 위치, 크기)}`

이미지 크기를 지정합니다. 크기 미지정 시 셀/범위에 맞춤이 기본값입니다.

| 크기 표기                 | 설명                 |
|-----------------------|--------------------|
| `fit` 또는 `0:0`        | 셀/범위에 맞춤 (기본값)     |
| `original` 또는 `-1:-1` | 원본 크기              |
| `200:150`             | 너비 200픽셀, 높이 150픽셀 |
| `200:-1`              | 너비 200픽셀, 높이 비율 유지 |
| `-1:150`              | 높이 150픽셀, 너비 비율 유지 |
| `0:-1`                | 셀 너비에 맞추고 높이 비율 유지 |
| `-1:0`                | 셀 높이에 맞추고 너비 비율 유지 |

```
${image(logo, B2)}            // 셀 크기에 맞춤 (기본값)
${image(logo, B2, fit)}       // 셀 크기에 맞춤
${image(logo, B2, original)}  // 원본 크기
${image(logo, B2, 200:150)}   // 200x150 픽셀
```

### 4.4 중복 마커

같은 이름, 같은 위치, 같은 크기를 가진 image 마커가 여러 개 있으면 중복으로 간주됩니다. 중복 마커가 발견되면 경고 로그가 출력되고 마지막 마커만 동작합니다.

- 크기가 다르면 중복이 아닙니다 (예: `${image(logo, B1:C2, 100:50)}`과 `${image(logo, B1:C2, 200:100)}`)
- 위치(`position`)가 없는 image 마커는 각각 마커 셀 위치에 삽입되므로 중복 체크 대상이 아닙니다
- 다른 시트에서 시트 접두사(`'Sheet1'!B1:C2`)로 같은 대상 위치를 참조하면 중복입니다

### 4.5 URL 이미지

이미지 데이터로 `ByteArray` 대신 HTTP(S) URL 문자열을 지정하면, Excel 생성 시점에 자동으로 다운로드하여 파일에 임베딩합니다. 생성된 Excel 파일을 열 때는 네트워크가 필요하지 않습니다.

```kotlin
val provider = simpleDataProvider {
    image("logo", logoBytes)                               // ByteArray
    imageUrl("banner", "https://example.com/banner.png")   // URL
}
```

**캐싱 동작**:
- 하나의 `generate()` 호출 내에서 같은 URL은 한 번만 다운로드됩니다 (설정과 무관)
- 여러 `generate()` 호출 간에도 캐싱이 필요하면 `imageUrlCacheTtlSeconds` 설정을 사용합니다 ([설정 옵션 참조](./configuration.md#imageurlcachettlseconds))

**실패 처리**: 다운로드 실패(네트워크 오류, 404 등) 시 경고 로그를 출력하고 해당 이미지를 건너뜁니다. Excel 생성 자체는 정상적으로 완료됩니다.

## 6. 자동 셀 병합

**문법**: `${merge(item.field)}`

repeat 확장 시 연속된 같은 값의 셀을 자동으로 병합합니다.
데이터가 병합 기준으로 사전 정렬되어 있어야 합니다.

- **DOWN repeat**: 세로 방향 병합
- **RIGHT repeat**: 가로 방향 병합

### 6.1 기본 사용

#### 템플릿

|   | A                    | B           | C           | D                              |
|---|----------------------|-------------|-------------|--------------------------------|
| 1 | 부서                   | 이름          | 직급          | ${repeat(employees, A2:C2, emp)} |
| 2 | ${merge(emp.dept)}   | ${emp.name} | ${emp.rank} |                                |

#### 데이터

```kotlin
mapOf(
    "employees" to listOf(
        mapOf("dept" to "영업부", "name" to "황용호", "rank" to "사원"),
        mapOf("dept" to "영업부", "name" to "한용호", "rank" to "대리"),
        mapOf("dept" to "개발부", "name" to "홍용호", "rank" to "과장"),
    )
)
```

#### 결과

<table>
  <tr><th></th><th>A</th><th>B</th><th>C</th></tr>
  <tr><td>1</td><td>부서</td><td>이름</td><td>직급</td></tr>
  <tr><td>2</td><td rowspan="2">영업부</td><td>황용호</td><td>사원</td></tr>
  <tr><td>3</td><td>한용호</td><td>대리</td></tr>
  <tr><td>4</td><td>개발부</td><td>홍용호</td><td>과장</td></tr>
</table>

> A2:A3이 "영업부"로 자동 병합됩니다.

### 6.2 다중 레벨 병합

여러 열에 각각 merge 마커를 사용하면 독립적으로 병합됩니다.

```
${merge(emp.dept)}   <- A열 (부서 기준 병합)
${merge(emp.team)}   <- B열 (팀 기준 병합)
${emp.name}          <- C열 (병합 없음)
```

### 6.3 주의 사항

- repeat 영역 밖에서 merge 마커를 사용하면 단순 값 치환만 수행됩니다.
- null 값은 병합 대상에서 제외됩니다.

## 8. 요소 묶음 (bundle)

**문법**: `${bundle(범위)}`

지정된 범위 내의 모든 요소를 하나의 묶음으로 취급하여, repeat 확장 시 묶음 전체가 일체로 이동합니다.

### 8.1 사용 목적

여러 열에 걸친 표(헤더, 데이터, 합계 행 등)가 있을 때, 일부 열의 repeat만 확장되면 표가 어긋날 수 있습니다. bundle로 표 전체를 묶으면 항상 일체로 이동합니다.

### 8.2 기본 사용

#### 템플릿

|     | A                             | B               | C    | D    | E      |
|-----|-------------------------------|-----------------|------|------|--------|
| 1   | ${repeat(depts, A2:B2, dept)} |                 |      |      |        |
| 2   | ${dept.name}                  | ${dept.revenue} |      |      |        |
| 3   | ${bundle(A4:E6)}              |                 |      |      |        |
| 4   | 이름                            | 매출              | 원가   | 이익   | 합계     |
| 5   | 황용호                        | 1000            | 500  | 500  | 2000   |
| 6   | 합계                            |                 |      |      | =SUM() |

- 1~2행: `depts` repeat이 A~B열에서 확장
- 4~6행: bundle로 묶인 표 (A~E열 전체)

아래는 depts가 3건일 때의 결과 비교입니다.

#### bundle이 없는 경우 -- 표가 어긋남

repeat 열 범위(A~B)만 밀리고, 범위 밖(C~E)은 원래 행에 그대로 남습니다.

|     | A      | B     | C    | D    | E      |
|-----|--------|-------|------|------|--------|
| 2   | Dept A | 52000 |      |      |        |
| 3   | Dept B | 38000 |      |      |        |
| 4   | Dept C | 28000 | 원가   | 이익   | 합계     |
| 5   |        |       | 500  | 500  | 2000   |
| 6   | 이름     | 매출    |      |      | =SUM() |
| 7   | 황용호    | 1000  |      |      |        |
| 8   | 합계     |       |      |      |        |

4행: A열은 "Dept C", C~E열은 헤더(원가/이익/합계) -- 같은 행에 서로 다른 내용이 들어가 표가 깨집니다.

#### bundle이 있는 경우 -- 표가 일체로 이동

bundle(A4:E6)으로 묶으면 A~E열 전체가 하나의 단위로 밀려, 모든 열이 같은 행에서 시작합니다.

|     | A      | B     | C    | D    | E      |
|-----|--------|-------|------|------|--------|
| 2   | Dept A | 52000 |      |      |        |
| 3   | Dept B | 38000 |      |      |        |
| 4   | Dept C | 28000 |      |      |        |
| 5   |        |       |      |      |        |
| 6   | 이름     | 매출    | 원가   | 이익   | 합계     |
| 7   | 황용호    | 1000  | 500  | 500  | 2000   |
| 8   | 합계     |       |      |      | =SUM() |

A~E열 모두 동일한 행에서 시작하므로 표가 정상적으로 유지됩니다.

### 8.3 제약사항

- **경계 걸침 금지**: 모든 범위 요소의 공통 규칙이 동일하게 적용됩니다 ([범위 경계 걸침 금지](#범위-경계-걸침-금지) 참조)
- **중첩 금지**: bundle 안에 다른 bundle을 배치할 수 없습니다.

## 9. 서식 유지

템플릿에 적용된 모든 서식은 생성된 Excel에 그대로 유지됩니다.

### 9.1 유지되는 서식

- 셀 정렬 (가로/세로)
- 글꼴 (이름, 크기, 굵기, 기울임, 밑줄, 색상)
- 채우기 색상
- 테두리
- 숫자 서식
- 조건부 서식

### 9.2 반복 영역 서식

반복 영역의 템플릿 행에 적용된 서식은 모든 반복 행에 동일하게 적용됩니다.
